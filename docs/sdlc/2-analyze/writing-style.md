# Writing Style

Documentation style for SumoLogic Query repository.

## Principles

- **Direct** - Bullets over prose
- **Scannable** - Readable in 30 seconds
- **High-level** - Principles, not implementation details
- **Short** - 20-40 lines per document

## Structure

```text
# Document Title

One-line description.

## Section Header

- Bullet point (concise)
- Another point
- Keep to 3-5 bullets per section
```

## What We Write

- Purpose and why
- Principles and patterns
- Key decisions

## What We Avoid

- Step-by-step tutorials (outdates quickly)
- Duplicating command references
- Over-detailed explanations
- More than ~200 lines per document

## Examples

- `docs/sdlc/2-analyze/naming-conventions.md` - Standards reference
- `docs/sdlc/3-design/overview.md` - Architecture overview
- `docs/sdlc/4-develop/rate-limiting.md` - Operational reference
